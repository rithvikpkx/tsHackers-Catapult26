## Core product loop

1. Ingest assignments
2. Correct effort (distortion profile)
3. Score failure risk
4. Intervene in calendar
5. Start mode smallest next step
6. Reflect on Pulse Board

## Why this scaffold matches

- Frontend pages are pre-split for Pulse Board, tasks, intervention, updated schedule, and start mode.
- Backend includes ingestion + intervention + calendar endpoints for Builder A scope.
- ML service exposes corrected effort and risk endpoints for Builder C ownership.
- Shared contracts freeze naming/schema early to avoid drift after hour 4.
